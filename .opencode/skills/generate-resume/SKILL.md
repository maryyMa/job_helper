## 功能说明

根据用户的个人经历和目标岗位JD，生成一份高度定制化的简历。

## 输入

- 用户个人经历（工作经历、项目经验、技能等）
- 目标岗位JD（可选，用于定制化）

## 输出

按照 [简历模板.md](./简历模板.md) 格式生成简历，并保存到指定路径。

### 输出路径

```
./output/{用户名字}/{面试公司}/{用户名字}-{目标公司}-{目标岗位}-简历.md
```

示例：`./output/张三/字节跳动/张三-字节跳动-高级Android工程师-简历.md`

### 简历结构

```markdown

|------|------|
```

## 执行规范

### 简历要求
1. 每个bullet point以动词开头（负责、主导、优化、搭建、推动）
2. 所有可量化的成果都要量化
3. 如有JD，自然融入JD中的核心关键词
4. 将与岗位最匹配的经历放在最显眼位置

### 篇幅限制（重要）
**目标：控制在1-2页A4纸（约600-1000字）**

1. **总字数**：简历正文不超过 **800字**
2. **个人简介**：精简为 **2句话**（职业定位+核心技术方向），删除套话
3. **技术栈**：只保留与岗位相关的技术，技能/工具用括号标注核心能力
4. **工作经历**：精简为 **1句话** 概述主要职责和方向，不展开细节
5. **项目经验**：展示 **2-5个** 与岗位最匹配的项目，不写时间线
   - 每个项目：技术栈 + 工作内容 + 成果
6. **删除模块**：
   - 核心数据表格 → 直接删除（融入具体描述）
   - 自我评价/个人总结 → 直接删除
   - 技能证书 → 只保留与岗位直接相关的
   - 培训经历 → 除非知名企业/权威认证，否则删除

### 内容真实性
1. 必须基于用户提供的真实经历，禁止编造
2. 量化数据如果用户未提供，基于行业合理推测并标注[预估]
3. 公司信息如需确认，使用webfetch访问官网

### 质量检查
- [ ] **篇幅是否控制在800字以内？**
- [ ] **项目经验是否2-5个？**
- [ ] 个人简介是否精简为2句话？
- [ ] 技术栈是否与岗位匹配？
- [ ] 工作经历是否精简为1句话概述？
- [ ] 项目经验是否体现技术深度？
- [ ] 是否融入了JD关键词？
- [ ] 是否删除了个人经历中没有的内容？
- [ ] 是否严格按照简历模板格式？
